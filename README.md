# Delivery Route Optimization

An end-to-end logistics analytics project focused on reducing delivery distance, estimated time, and operational cost through route optimization, geospatial visualization, and predictive modeling.

This project simulates a delivery operation and applies analytics, optimization, and machine learning techniques to improve decision-making in last-mile logistics.

The central business question behind this project is:

**How can route optimization reduce delivery distance, delivery time, and operational cost while helping identify deliveries at higher risk of delay?**

This case study was designed as a portfolio project to demonstrate how data analytics, operations research, and machine learning can be combined to solve a realistic logistics problem.


## Business Problem

Delivery operations often face inefficiencies caused by:

- poor route planning,
- unnecessary travel distance,
- traffic-related delays,
- and rising transportation costs.

These inefficiencies can directly affect:

- operational efficiency,
- customer satisfaction,
- delivery reliability,
- and total logistics cost.

This project addresses that challenge by combining synthetic data generation, exploratory analysis, route optimization, route visualization, and delay prediction into one end-to-end workflow.


## Key Results

Below are the main performance indicators generated by the project.

- **Original route distance:** `8376.8 km`

## Methodology

The project is structured in six main stages:

1. **Data Simulation**  
   Create a synthetic delivery dataset with operational variables such as city, coordinates, distance, traffic level, vehicle type, delivery window, cost, and delay risk.

2. **Exploratory Data Analysis**  
   Explore distance, time, cost, and delay-related patterns through analytical summaries and charts.

3. **Route Optimization**  
   Use Google OR-Tools to calculate an optimized route and compare it with the original route sequence.

4. **Geospatial Visualization**  
   Create interactive maps for both original and optimized routes using Folium.

5. **Delay Prediction**  
   Train a machine learning model to predict whether a delivery is likely to be delayed.

6. **Business Reporting**  
   Consolidate final metrics and charts to support portfolio presentation and business interpretation.

## Implemented Scripts

### `src/data_generation.py`
Generates the synthetic delivery dataset and saves the raw file to `data/raw/`.

### `src/eda.py`
Cleans the dataset, builds summary metrics, and generates exploratory charts.

### `src/optimization.py`
Uses Google OR-Tools to calculate an optimized route for a subset of deliveries.

### `src/mapping.py`
Creates interactive Folium maps for the original and optimized delivery routes.

### `src/modeling.py`
Builds a machine learning model to predict delivery delays and generates classification outputs.

### `src/reporting.py`
Consolidates route, operational, and model outputs into presentation-ready figures and metrics.

### `src/run_pipeline.py`
Runs the full project pipeline in sequence.

## How to Run

After cloning the repository, install the required dependencies:

```bash
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

Run the full project pipeline:

```bash
.\.venv\Scripts\python src\run_pipeline.py
```

Suggested execution order for individual scripts:

1. `src/data_generation.py`
2. `src/eda.py`
3. `src/optimization.py`
4. `src/mapping.py`
5. `src/modeling.py`
6. `src/reporting.py`
